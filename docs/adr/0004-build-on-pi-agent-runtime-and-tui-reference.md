# Build on Pi Agent Runtime and Defer TUI

The Daily Brief Agent MVP will be built around the `earendil-works/pi` agent runtime concepts, with the Agent owning orchestration, tool execution, state management, and event streaming even though the first product slice could be implemented as a simpler pipeline. This is intentional because practicing Agent architecture is part of the project's purpose. The MVP will use an Operational CLI rather than a chat-based Control TUI; a future V2 Control TUI may reference `earendil-works/pi` TUI patterns for terminal interaction and rendering, but it must not own Daily Brief business logic or skill routing. We will consume pi through pinned npm package dependencies rather than vendoring the source or adding it as a submodule unless a future patching requirement justifies that cost.
